ebgg

to load a background, go to the main menu and click on "load a background".

from there, you can choose any background from the list of built-in backgrounds that comes with ebgg. backgrounds come in the .deb format.

if you would like to import a background somebody else has made, ask them for the .deb file, and put it in the "backgrounds" folder, inside of the processing sketch folder.

to edit a background, go to the main menu and click on "editor". you will be greeted with a bunch of options:

name: the name of the background. this property will define the name of the saved file, so make sure to change it to avoid overwriting the existing background.
palette: the color palette used. it is a list of colors in hexadecimal format. the "paloffset >" label (that may move depending on the background) is the palette index offset.
palette map: the "image" used by the background: the pattern that gets filtered then drawn repeatedly.
switch pal colors every n frame: number that defines how often the palette offsets every frame. a frame is 1/60th of a second, two frames are 1/30th of a second, etc.
switch pal colors?: yes-no property for choosing either to offset the palette or not.
switch pal colors in reverse?: yes-no property for choosing either to offset the palette in reverse or not.
dont switch the first n items: number that defines which first palette items to stop offsetting before rendering.
palette switch multiplier: how much does the palette offset change per frame.
camera x velocity: how much the camera's x position moves per frame.
camera y velocity: how much the camera's y position moves per frame.
scale: how much the rendered output is scaled in. the more it's high, the more it gets zoomed in.
x wavyness scale: how much the x position gets a wavy/sine effect.
x wavyness frequency: how much the x position wave moves every frame.
interleaved x wavyness?: yes-no property for choosing whether the x position wave is interleaved or not.
y wavyness scale: how much the y position gets a wavy/sine effect.
y wavyness frequency: how much the y position wave moves every frame.
x static effect: how much the x position gets offset by a random value for every pixel. makes the background shake, or look like analog tv snow.

if you would like to edit numbers more faster, some properties allow use of "big steps". press the A key to substract, and D to add.
